RULES

разработка ведется по staged подходу

Stage 1–3 уже частично реализованы

нельзя:

ломать текущий pipeline n8n
переписывать архитектуру с нуля
удалять существующие таблицы без миграции
добавлять сложные enterprise решения
вводить kafka или сложные брокеры
добавлять UI на текущем этапе

нужно:

использовать postgres как source of truth
использовать docker compose
делать небольшие безопасные изменения
использовать repo-first подход
использовать document-first подход
использовать inspect-before-change

все изменения:

обратимо применимы
миграционно совместимы
не ухудшают доставку событий

каждое событие должно иметь lifecycle

каждый файл должен иметь статус обработки

retry обязателен

исключить:

потерю файлов
потерю событий
дубли отправки
отправку не тому получателю

env:

используется единый файл:

env/app.env

single source of truth for secrets

docker compose
python services
